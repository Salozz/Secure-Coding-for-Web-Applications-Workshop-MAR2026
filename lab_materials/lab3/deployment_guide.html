🚀 Lab 3 — Secure Deployment

Deploy to the Internet

Push your patched code to GitHub, deploy to Netlify, verify your security headers score A+.

✅ Before You Start — Checklist

index.html — uses textContent not innerHTML (Lab 1A fixed)
index.html — XSS payload now appears as text, does not execute
_headers file exists in the root of your project folder (same level as index.html)
You have a GitHub account and are logged in at github.com
You have a Netlify account and are logged in at app.netlify.com
Git is installed — run git --version in the terminal to confirm
1

Initialise Git in Your Project Folder

Open VS Code terminal: View → Terminal (or Ctrl+`). Make sure you are inside your project folder.

git init # Expected: Initialized empty Git repository in .../my-project/.git/
git add . # Stages all files for commit. The dot means "everything in this folder"
git status # Shows files in green = ready to commit
git commit -m "Secure web app: fixed XSS, SQLi, CSRF, headers" # Creates a snapshot. You'll see: [main (root-commit) abc1234]
2

Create a GitHub Repository and Push

  1. Go to github.com → click + (top right) → New repository
  2. Name: secure-web-app · Visibility: Public · Do NOT tick README/gitignore/license
  3. Click Create repository. Copy the HTTPS URL shown (looks like https://github.com/YOUR_NAME/secure-web-app.git)
  4. Back in VS Code terminal:
git remote add origin https://github.com/YOUR_USERNAME/secure-web-app.git # Replace YOUR_USERNAME with your actual GitHub username
git branch -M main # Renames branch to 'main' (modern convention)
git push -u origin main # Uploads your code to GitHub. May ask for login. # If password fails: use a Personal Access Token instead (see troubleshooting)

✅ Refresh github.com/YOUR_USERNAME/secure-web-app — your files should appear.

3

Deploy on Netlify

  1. Go to app.netlify.com → Log in
  2. Click Add new site → Import an existing project → Deploy with GitHub
  3. Authorise Netlify → select your secure-web-app repository
  4. Build settings: leave all fields blank (static HTML, no build needed)
  5. Click Deploy site — wait ~30 seconds
  6. Netlify gives you a URL: e.g. brave-coder-abc123.netlify.app
  7. Click the URL — your site is LIVE with HTTPS!
4

Verify Your Security Headers

  1. Copy your Netlify URL (e.g. brave-coder-abc123.netlify.app)
  2. Go to securityheaders.com
  3. Paste your URL → click Scan
  4. Target: A or A+ grade
  5. Share your URL and grade in the class chat! 🎉

If you get B or lower: check your _headers file is in the root of the project folder (not in a subfolder), and that it has no file extension.

🔧 Troubleshooting

Error: "git: command not found"
Git is not installed. Download from git-scm.com, install it, then restart VS Code. OR skip to the drag-and-drop path below.
Error: "Please tell me who you are"
Run these two commands, then try again:
git config --global user.email "your@email.com"
git config --global user.name "Your Name"
Error: "Authentication failed" when pushing
GitHub no longer accepts account passwords for git push. Create a Personal Access Token: GitHub → Settings → Developer settings → Personal access tokens → Tokens (classic) → Generate new token → tick "repo" → Generate. Use this token as the password.
securityheaders.com gives grade B or C
Your _headers file may not be in the right place. It must be in the project root (same folder as index.html), named exactly _headers with no .txt or other extension. Re-deploy after moving it.

🔄 Backup Path — Deploy without Git

If Git isn't working, you can still deploy directly:

  1. Go to app.netlify.com
  2. On the dashboard, scroll down to the section that says "Want to deploy a new site without connecting to Git?"
  3. Drag your entire project folder (not individual files) onto the dashed rectangle
  4. Netlify deploys in ~5 seconds and gives you a live URL immediately
  5. The _headers file must be inside the folder before you drag it
