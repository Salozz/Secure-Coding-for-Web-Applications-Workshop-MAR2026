Initialise Git in Your Project Folder

Open the VS Code terminal: View → Terminal (or press Ctrl+`). Check the terminal prompt shows your project folder.

git init # Expected: Initialized empty Git repository in .../my-project/.git/ git add . # The dot ( . ) stages ALL files in this folder git status # Files shown in green = staged and ready to commit git commit -m "Secure web app: fixed XSS, SQLi, CSRF, headers" # Creates a snapshot. You'll see: [main (root-commit) abc1234]
2

Create a GitHub Repository and Push

  1. Go to github.com → click + (top right) → New repository
  2. Name: secure-web-app  ·  Visibility: Public  ·  Do NOT tick any initialisation checkboxes (no README, no .gitignore, no licence)
  3. Click Create repository. Copy the HTTPS URL shown — e.g. https://github.com/YOUR_NAME/secure-web-app.git
  4. Back in the VS Code terminal:
git remote add origin https://github.com/YOUR_USERNAME/secure-web-app.git # Replace YOUR_USERNAME with your actual GitHub username git branch -M main # Renames the branch to 'main' — the modern default git push -u origin main # Uploads your code to GitHub. May ask for login. # Success: Branch 'main' set up to track remote branch 'main' from 'origin'

✅ Refresh github.com/YOUR_USERNAME/secure-web-app — all your files should appear.

3

Deploy with Cloudflare Pages

📌 About the _headers file on Cloudflare Pages
Cloudflare Pages supports the exact same _headers file format. Place it in your project root and Cloudflare reads it automatically — no extra configuration needed. Your security headers will be live from the first deployment.
→ Go to dash.cloudflare.com and log in to your Cloudflare account.
→ In the left sidebar, click Workers & Pages.
→ Click Create application button → click Connect with GitHub.
→ Select your GitHub Account. Authorise Cloudflare to access your GitHub account when prompted.
→ Select your web-app repository from the list. Click Next to set up your application.
→ Build settings — leave all three fields blank or as shown:
Project name → your domain name
Build command → (leave empty)
Deploy command → npx wrangler deploy
Builds for non-production branches → tick to enable
Our site is plain HTML — no advanced settings are needed.
→ Click Deploy. Watch the deployment log — takes about 20–40 seconds.
→ When done, Cloudflare gives you a URL like: {domain}.{subdomain}.workers.dev
Click it — your site is LIVE on the internet with HTTPS! 🎉
🔁 Future updates are automatic. Every time you git push to GitHub, Cloudflare Pages detects it and re-deploys within ~1 minute. No need to log into Cloudflare again.
4

Verify Your Security Headers

  1. Copy your Cloudflare Pages URL — e.g. {domain}.{subdomain}.workers.dev
  2. Go to securityheaders.com
  3. Paste your URL → click Scan
  4. Target: A or A+ grade
  5. Share your URL and grade in the class chat! 🎉

If you get B or lower: check your _headers file is in the project root (not in a subfolder) and has no file extension. Re-commit and push — Cloudflare will auto-redeploy.

🔁 Updating Your Site After Deployment

Once Cloudflare Pages is connected to your GitHub repository, every push triggers a new deployment automatically.

git add . git commit -m "Update: describe your change" git push # Cloudflare Pages detects the push and redeploys in ~1 minute # No need to log into Cloudflare — it's fully automatic

🔧 Troubleshooting

Error: "git: command not found" or "git is not recognised"
Git is not installed. Download from git-scm.com, install it, then restart VS Code. If you can't install it right now, use the drag-and-drop backup path below.
Error: "Please tell me who you are"
Run these two commands, then retry the commit:
git config --global user.email "your@email.com"
git config --global user.name "Your Name"
Error: "Authentication failed" when pushing to GitHub
GitHub no longer accepts your account password for git push. Create a Personal Access Token: GitHub → profile picture → Settings → Developer settings (bottom of left sidebar) → Personal access tokens → Tokens (classic) → Generate new token → tick the repo scope → Generate. Use this token as the password.
Cloudflare Pages build failed
Make sure Build command and Build output directory are both empty in the Pages settings. Our site is static HTML — no build step. To fix: go to your Pages project → Settings → Builds & deployments → clear both fields → Retry deployment.
securityheaders.com gives grade B, C, or F
Check these in order: (1) The file is named exactly _headers — no .txt, no extension of any kind. (2) The file is in the project root — same folder as index.html, not inside a subfolder. (3) Commit and push again — Cloudflare will redeploy with the corrected file.
Error: "nothing to commit" when running git commit
All your changes are already saved. Make sure you saved files in VS Code first (Ctrl+S on each open file), then run git add . again before committing.

🔄 Backup Path — Deploy Without Git (Direct Upload)

If Git isn't working, Cloudflare Pages supports direct folder upload:

  1. Go to dash.cloudflare.com → Workers & Pages → Create application → Pages
  2. Instead of "Connect to Git", click Upload assets (the Direct Upload option)
  3. Give your project a name, e.g. secure-web-app
  4. Drag your entire project folder (not individual files) onto the upload area — make sure the _headers file is inside the folder
  5. Click Deploy site — you get a live URL in seconds

Note: with direct upload, future changes require re-uploading manually. The GitHub-connected path is recommended for ongoing work.
