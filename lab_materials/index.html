Quick Reference

XSS — Cross-Site Scripting
User input rendered as HTML → attacker's script runs in victim's browser
textContent not innerHTML
SQL Injection
User input concatenated into SQL query → attacker modifies query logic
parameterised queries
CSRF — Request Forgery
Browser auto-sends cookies → attacker triggers requests without user intent
CSRF token + SameSite
Insecure Authentication
Weak hashes, predictable tokens, or missing cookie flags → account takeover
bcrypt + CSPRNG + flags
Security Misconfig
Debug on, default creds, missing headers → attacker reads config
_headers + debug off

⚡ Lab Attack Payloads

XSS — onerror trigger
<img src=x onerror="alert('XSS: '+document.cookie)">
Paste into the comment input in Lab 1A. The broken image src fires onerror → JavaScript runs.
XSS — page takeover
<img src=x onerror="document.body.style.background='red'">
Visual proof that injected JS runs. Changes the whole page background.
SQL Injection — bypass login
admin' OR '1'='1' --
Type in the username field in Lab 1B. The -- comments out the password check.
SQL Injection — password field
' OR '1'='1
Type in the password field. Same logic — makes the WHERE clause always true.
