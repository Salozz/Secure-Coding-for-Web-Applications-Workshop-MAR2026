When ON: the bank checks that the transfer request includes the correct token. The attacker's form has no token — the request is rejected.

📄 The Fix: Adding CSRF Token to Forms

<!-- ✅ Secure transfer form with CSRF token -->
<form action="/transfer" method="POST">

  <!-- Server generates this per session and embeds it -->
  <input type="hidden" name="csrf_token"
         value="a3f8b2c1d9e4f6..."> <!-- long random string -->

  <input type="text"   name="to"     placeholder="Recipient">
  <input type="number" name="amount" placeholder="Amount">
  <button type="submit">Transfer</button>
</form>

// Server-side check (Node.js example):
if (req.body.csrf_token !== req.session.csrf_token) {
  return res.status(403).send('CSRF check failed — request rejected');
}
// Only proceed if token matches

// Also set cookies with SameSite=Strict:
// Set-Cookie: session=abc123; SameSite=Strict; HttpOnly; Secure
// With SameSite=Strict: cross-site requests carry NO cookie.
// Attacker's form gets no session → bank rejects as unauthenticated.
