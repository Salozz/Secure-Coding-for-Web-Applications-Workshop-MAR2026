✅ What was fixed

Fix 1: Replaced innerHTML += with textContent — user input is now treated as literal text, never parsed as HTML.

Fix 2: Added a Content-Security-Policy meta tag in <head> — even if injection slips through, the browser refuses to run inline scripts.

📄 The Fixed Code

function postComment() {
  const raw = document.getElementById('commentInput').value.trim();
  if (!raw) return;

  // ✅ FIX: createElement + textContent
  //    textContent NEVER parses input as HTML.
  //    Angle brackets become visible characters, not tags.
  const div = document.createElement('div');
  div.className = 'comment-item';
  div.textContent = raw;
  document.getElementById('comments').appendChild(div);

  document.getElementById('commentInput').value = '';
}
