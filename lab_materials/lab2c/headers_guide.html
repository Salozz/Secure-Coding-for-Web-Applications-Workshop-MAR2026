🛡️ Lab 2C — Security Headers

HTTP Security Headers

Six headers that dramatically improve your site's security — takes 5 minutes to add.

📋 Your Tasks

  1. In your project folder, create a file named exactly: _headers (underscore, no extension).
  2. Copy the content from the code block below into that file.
  3. Save it in the root of your project folder — same level as index.html.
  4. Deploy to Cloudflare Pages (Lab 3). After deployment, go to securityheaders.com and scan your live URL.
  5. Target grade: A or A+. If you get lower, check the file is in the root folder with no extension.
  6. Read the headers table below — for each header, know what attack it stops.

  Content-Security-Policy: default-src 'self'; script-src 'self'; style-src 'self' 'unsafe-inline'; img-src 'self' data:
  X-Frame-Options: DENY
  X-Content-Type-Options: nosniff

📖 What Each Header Does

HEADER
WHAT IT DOES
WITHOUT IT...
Content-Security-Policy
Stops
Tells the browser which domains can serve scripts, styles, and images. Inline scripts and external scripts from unknown domains are blocked by default.
XSS payloads execute even if they slip through your textContent fix. Any injected <script> or event handler runs freely.
X-Frame-Options: DENY
Stops
Prevents your page from being loaded inside an <iframe> on any other domain. Your site cannot be "framed" by another page.
Clickjacking: attacker overlays your site in a transparent iframe, tricks users into clicking invisible buttons (e.g. "Confirm Purchase").
X-Content-Type-Options: nosniff
Stops
Tells the browser to trust the Content-Type the server declares. Do not guess or "sniff" file types.
A malicious file uploaded as an image (image/png) could be executed as JavaScript if the browser decides to sniff the content type.
Strict-Transport-Security
Stops
Forces HTTPS for your domain. Even if a user types http://, the browser upgrades to https:// automatically. max-age=31536000 = 1 year.
SSL stripping: attacker on the same network downgrades your connection to plain HTTP, intercepts all traffic including session cookies.
Referrer-Policy: no-referrer
Stops
Controls how much of your URL is sent in the Referer header when users navigate to other sites. no-referrer sends nothing.
Sensitive URL parameters (password reset tokens, session IDs in URLs) are leaked to third-party sites in the Referer header.
Permissions-Policy
Stops
Restricts which browser APIs the page can use. camera=() means NO origin can use the camera. Also controls mic, geolocation, payments.
Even if XSS fires on your page, the attacker could activate the user's camera or microphone. This header prevents that.

🚀 Deployment Steps

1

Create _headers in project root

Same folder as index.html. No file extension. Exactly: _headers

2

Commit & push to GitHub

git add . → git commit -m "add security headers" → git push

3

Cloudflare auto-deploys

Cloudflare Pages detects the push and redeploys your site in ~1 minute. Your live URL updates automatically.

4

Verify at securityheaders.com

Paste your yoursite.pages.dev URL. Check you get an A or A+ grade.

A+

Target grade

What the grade means

securityheaders.com scans your live site's HTTP response headers and grades them A+ through F. An A+ means all major security headers are present and correctly configured. The most common reasons for a B or below: missing CSP, missing Permissions-Policy, or the _headers file not being found by Cloudflare Pages (check it's in the project root with no file extension).
